## Summary
Information Bottleneck Option Learning (IBOL) is an unsupervised, reward-free skill discovery method which uses a pretrained "**linearizer**" policy with an "**information bottleneck (IB)**" objective over trajectory encodings. 
Linearizer $\pi^{\text{lin}}(a_t|s_t, g_t)$ is a goal-conditioned low-level policy which outputs a probability distribution over actions in the direction of a given goal. 

## Prior works

### Latent-First Methods
Latent-First methods sample a skill variable $z$ first, then run the agent conditioned on that $z$, then train a discriminator to recover $z$ from the resulting states.

1. DIAYN (Diversity Is All You Need)
DIAYN is an information-theoretic skill discovery method which extends VIC by discriminating on **every state** along the trajectory (instead of the final state of the trajectories). It uses an intrinsic reward $r_t = \log q_\phi (z \mid s_t) - \log p(z)$ where $p(z)$ is an uniform prior over discrete or Gaussian prior over continuous $z$. 
Since it discriminates on raw states, unimportant dimensions (e.g., joint velocities, contact forces) dominate the signal, and therefore discriminator tends to discriminate the state based on these features which lead to producing visually entangled skills. 

2. VALOR (Variational Option Discovery Algorithms)
VALOR uses a **trajectory-level** decoder $p_D(z \mid s_{0:T})$ rather than a per-state discriminator, reconstructing the original $z$ from the full-trajectory.

3. DADS (Dynamics-Aware Unsupervised Discovery of Skills)
DADS learns a **skill dynamics model** $q(s_{t+1} \mid s_t, z)$ that predicts next states given current state and skill. The intrinsic reward is $r_z(s_t, a_t, s_{t+1}) = \log \frac{q_\phi (s_{t+1} \mid s_t, z)}{\frac{1}{L} \sum_{i=1}^L q_\phi(s_{t+1} \mid s_t, z_i)} + \log L$. While the learned dynamics model enables model-based planning at test time, skills are defined purely by transition dynamics, not by an interpretable latent space.

### Trajectory-First Methods
Traject-First methods first collect trajectories with some exploration policy, encode each trajectory into a skill latent $z$ using a VAE, then train a skill policy to reproduce trajectories given their $z$.

1, SeCTAR (Self-Consistent Trajectory Autoencoder)
It uses an exploration policy that maximizes trajectory entropy $h(p(\tau))$, a VAE encoder to compress trajectories into $z$, and a skill policy to reconstruct trajectories from $z$. 
SeCTAR has a limitation in that the exploration policy might generate trajectories that are diverse but unstructured. 

2. EDL (Explore, Discover, Learn)
Similar to SeCTAR, but it works at the **state level** rather than trajectory level. Since it uses a state, it discards a temporal structure (e.g., two trajectories that visit the same states in different orders get the same $z$). In addition, state entropy maximization is a weak proxy for skill diversity; in other words, covering many states doesn't mean that the skills are structured or interpertable.
