## Problem
Given an integer N, find the minimum number of operations to reduce N to 1, where you may divide by 3 (if divisible), divide by 2 (if divisible), or subtract 1.

## Approach
- Initial thought: try a greedy "always divide by the largest possible divisor" — fails on small counterexamples (e.g. N = 10).
- Where I got stuck: realizing that the optimal choice at each step depends on the optimal results of smaller subproblems, not just the current number.
- Final approach: bottom-up DP. For each `i` from 2 to N, compute `dp[i]` as the minimum of `dp[i-1] + 1`, and (if divisible) `dp[i/2] + 1` or `dp[i/3] + 1`.

## Solution
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    int n;
    cin >> n;

    vector<int> dp(n + 1, 0);
    for (int i = 2; i <= n; i++) {
        dp[i] = dp[i - 1] + 1;
        if (i % 2 == 0) dp[i] = min(dp[i], dp[i / 2] + 1);
        if (i % 3 == 0) dp[i] = min(dp[i], dp[i / 3] + 1);
    }

    cout << dp[n] << endl;
    return 0;
}
```

## Complexity
- Time: O(n)
- Space: O(n)

## Key Takeaway
When a greedy choice can be invalidated by later constraints, check whether the problem has overlapping subproblems — if so, bottom-up DP over all smaller values is usually simple and fast enough.
